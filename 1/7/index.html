Sketching with Embeddings

2025-01-07

Main-Quest

Today’s objective was to study the relationship between the embeddings created yesterday. One used only video annotations in order to embed points in 2D space, while the other wvisualized the same points by learning on a completely different (and much larger), dataset of a similar nature.

What I’m studying here is in some sense how good a dimension-reduction model can get when trained on one dataset and used for another at inference-time. I’m hoping that turning two collections of points into a collection of lines will be visually interesting.

So I loaded up the results from yesterday and whipped up a quick “version 0” of the image:

import matplotlib.pyplot as plt
import numpy as np

caption_embed = np.load("dataXtrain.npy")
kosmos_embed = np.load("dataXdata.npy")

# draw a line segment between each pair of points between the two embeddings
assert caption_embed.shape[0] == kosmos_embed.shape[0]
num_points = caption_embed.shape[0]
fig, ax = plt.subplots(figsize=(10, 10))
for i in range(num_points):
    ax.plot([caption_embed[i, 0], kosmos_embed[i, 0]], [caption_embed[i, 1], kosmos_embed[i, 1]], color="black", alpha=0.01)
ax.axis("off")
fig.tight_layout(pad=0)
fig.savefig("connect-embed.svg")
fig.show()  # yes, I used a notebook today for some reason

I can see a lot of “collapsing” behavior going on: there’s a consolidation of the data around two main regions of high-density. Let’s add some labels (red circles) to the original embedding (the one that was not exposed to almost half a million human-annotated captions):

Aesthetically it’s an okay start, but I think I can do a little better. I do like the bokeh-like effect that the red circles have. I think I can emphasize that more by making the point size correspond to the distance traveled.

distances = np.linalg.norm(caption_embed - kosmos_embed, axis=1)
d = 10 * np.log(distances + 1)
# ...
ax.scatter(kosmos_embed[:, 0], kosmos_embed[:, 1], color="red", alpha=0.1, s=d)
# ...

Also - instead of drawing a straight line from point-to-point, I can interpolate a curve so there’s a bit more going on (and introduce a bit of randomness, which is a theme I’ve been incorporating into my art since 2011).

With random curves between each pair of points, the balance between white and red could use further adjustment.

The next steps will be to generate a few more variants using the same data and processing pipeline, changing only the random initialization state of the dimension-reduction algorithm. Such an exercise would give me a better idea of the visuals possible and whether some aesthetic qualities show up consistently as a result of structure being discovered in the data. And of course, the exploration will be in search of something visually compelling that I may be interested in printing.

Personally, I prefer the original variant with straight lines and equal-sized red circles. Maybe playing with opacity instead of size will help emphasize the “bokeh effect” that I found promising.
