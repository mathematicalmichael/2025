Finding Semantic Similarity in Text

2025-01-03

Taking a brief detour from image-to-text models, I wanted to get ahead of some downstream dependencies for the projects I have in mind. Namely, I want to make sense of the relationships between different sequences of text.

In the previous two posts, it was clear that there was a lot of instability in the bounding boxes that were detected in each frame of video. What I aim to visualize is the “path through semantic space” that these annotations take over time.

The exercise is in compressing time into a single image.

I have a number of projects that require visualizing temporal data as an image. Training models takes time, and I already have a working prototype of Step 1 of my pipeline. I thought it would be wise to “work in broad strokes” instead of getting caught up in details prematurely, so I wanted to get ahead of acquiring the data I would need for Step 3.

I found several hundred thousand human-annotated images courtesy of Google.

With some sample data acquired, I can now move on to sentence-embedding, which will take a string and turn into a list of numbers.

Let’s adopt a basic example script from HuggingFace’s Hubnomic-embed-text-v1.5:

# txt2vec.py
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("nomic-ai/nomic-embed-text-v1", trust_remote_code=True, device='cuda')
sentences = ['this is a sentence that will become a long list of numbers']
embeddings = model.encode(sentences)
print(embeddings)

(Note the typo in their example code… v1 should be v1.5 which I only noticed days later and fixed references to in my posts).

Time to run it on some data and investigate what is inside of these files I just downloaded! First let’s get the strings into a text file:

data.txt:
        cat open_images_validation_localized_narratives.jsonl | jq .caption > data.txt

And with some slight modifications of the example script, we have our vector representations:

from sentence_transformers import SentenceTransformer
import numpy as np

with open('data.txt', 'r') as f:
    data = f.readlines()

model = SentenceTransformer("nomic-ai/nomic-embed-text-v1", trust_remote_code=True, device='cuda')
embeddings = model.encode(data)
np.save('embed.npy', embeddings)

And finally, I can visualize the results with some plotting code that picks up the data I just saved:

from pathlib import Path

import matplotlib.pyplot as plt
import numpy as np
from pacmap import PaCMAP
from pacmap.pacmap import load, save

data = np.load("embed.npy")

if Path("pcmp.joblib").exists():
    reducer = load("pcmp")
else:
    reducer = PaCMAP(random_state=21, save_tree=True)
    reducer.fit(data)
    save(reducer, "pcmp")

X = reducer.transform(data)

fig, ax = plt.subplots(figsize=(10, 10))
ax.scatter(X[:, 0], X[:, 1], c="k", s=0.1, alpha=0.2)
ax.axis("off")
ax.set_aspect(1)
fig.savefig("embed.svg", dpi=300, bbox_inches="tight")

Nothing above was a new process in my toolbox, but I wanted to write the scripts from scratch and document the basic outline end-to-end, without re-using previous code. This is a pattern I have used many times in my job in order to process text data, and is a common practice in the data science community. Models can be switched out in any step, including the sentence-embedding model and the dimensionality reduction model that were used here.

Now with some minor modifications, I’ll be able to plot the path that a point takes over time as an image, in a space that clearly groups sentences of similar semantic meaning close together.
